# Exploratory Terms & Details for Newbies

Below is a curated list of key concepts, terminology, and learning resources. Each entry includes a brief description and links for further self-study.

| Term                  | Description                                                                                         | Learn More                                                      |
|-----------------------|-----------------------------------------------------------------------------------------------------|-----------------------------------------------------------------|
| **Node.js**           | JavaScript runtime built on Chrome’s V8 engine for writing server-side code.                        | https://nodejs.org/en/docs/                                    |
| **Express**           | Minimalist web framework for Node.js to build APIs and web applications.                            | https://expressjs.com/en/starter/installing.html                |
| **REST**              | Architectural style for designing networked applications using stateless HTTP methods.              | https://restfulapi.net                                         |
| **CRUD**              | Acronym for Create, Read, Update, Delete—the basic operations on data.                              | https://en.wikipedia.org/wiki/Create,_read,_update_and_delete   |
| **ES Modules**        | Standardized JavaScript module system using `import` and `export`.                                   | https://nodejs.org/api/esm.html                                 |
| **Middleware**        | Functions that process requests in Express before reaching route handlers, used for logging, auth, and parsing. | https://expressjs.com/en/guide/using-middleware.html            |
| **In-Memory Store**   | Data stored in application memory (e.g., an array). Resets on server restart—used for prototypes.   | https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array |
| **Environment Variables** | Configuration values (e.g., `PORT`, database URIs) stored outside code for flexibility across environments. | https://www.npmjs.com/package/dotenv                           |
| **HTTP Methods**      | Common verbs: GET, POST, PUT, DELETE that define actions on resources.                               | https://developer.mozilla.org/docs/Web/HTTP/Methods             |
| **JSON**              | JavaScript Object Notation—a lightweight data-interchange format used in APIs.                      | https://www.json.org/json-en.html                               |
| **CORS**              | Cross-Origin Resource Sharing—a browser security feature controlling requests across domains; handled via HTTP headers. | https://developer.mozilla.org/docs/Web/HTTP/CORS                |
| **Supertest**         | Library for testing Node.js HTTP servers.                                                           | https://github.com/visionmedia/supertest                       |
| **Nodemon**           | Utility that automatically restarts Node.js application on file changes.                            | https://www.npmjs.com/package/nodemon                          |
| **Git & GitHub**      | Version control and hosting platform for collaboration using branches and pull requests.            | https://docs.github.com/en/get-started                          |
| **ESLint & Prettier** | Tools for linting (code style checking) and formatting JavaScript code consistently.                | https://eslint.org/  <br> https://prettier.io/                  |
| **Arrow Functions**   | Concise syntax for function expressions in JavaScript (`() => {}` ).                                 | https://developer.mozilla.org/docs/Web/JavaScript/Reference/Functions/Arrow_functions |
| **Async/Await**       | Syntax for handling asynchronous operations in JS, built on promises for cleaner code.              | https://developer.mozilla.org/docs/Learn/JavaScript/Asynchronous/Async_await |
| **Array Methods**     | Built-in functions like `.map()`, `.filter()`, `.reduce()` for processing arrays.                    | https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/map |
| **Template Literals** | String literals allowing embedded expressions with backticks (`` `Hello ${name}` ``).                | https://developer.mozilla.org/docs/Web/JavaScript/Reference/Template_literals |
| **Request Validation**| Checking incoming request data for required fields and types before processing.                     | https://express-validator.github.io/docs/                       |
| **Modularization**    | Breaking code into separate modules/files to improve organization and reuse.                        | https://nodejs.dev/learn/nodejs-module-system                  |
